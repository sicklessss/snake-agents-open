# Snake Agents 完整游戏规则

## 一、游戏场地

| 参数 | 值 |
|------|-----|
| 网格大小 | 30×30 格子 |
| Tick 频率 | 50ms/tick（线上当前配置约每秒 20 帧） |
| 每局时长 | 180 秒（3 分钟） |

**10 个出生点**（坐标, 朝向）：
(5,5)→右, (25,5)→左, (5,25)→右, (25,25)→左, (15,3)→下, (15,27)→上, (3,15)→右, (27,15)→左, (10,10)→右, (20,20)→左

出生时与其他蛇头的曼哈顿距离必须 ≥5 格。

---

## 二、房间类型

### Performance（练习房）
- 最多 6 间（A-F），根据活跃 agent 数量自动创建；每满 10 个 bot 就预开下一间（0-9 个开 1 间，10-19 个开 2 间，以此类推）
- 每间最多 10 条蛇
- 自动用 Normal Bot 填满空位

### Competitive（竞技场）
- 固定 1 间 (competitive-1)
- 当前固定 10 条蛇
- 含**障碍物系统**和**食物递减**（Performance 房没有）

#### 竞技场 Bot 参赛规则
- **自动填充**：每轮 COUNTDOWN 阶段，系统自动从已注册的 agent bot 池中**随机抽取**填满 10 个位置
- **填充优先级**：① 付费入场的 bot → ② 随机选取的 agent bot → ③ Normal Bot 补位
- **参赛条件**：`botType=agent` + 有脚本 + 已注册（unlimited）或有剩余 credits
- **可同时参加 performance 和 competitive**：同一个 bot 可以在两种房间各开一个独立 worker
- **每轮随机洗牌**：不是固定阵容，每局 COUNTDOWN 都会重新随机抽取
- **付费入场**：任何钱包可以为任何 bot 支付 0.001 ETH 锁定竞技场席位（优先于随机填充）

---

## 三、比赛流程

```
COUNTDOWN (30秒) → PLAYING (180秒) → GAMEOVER (30秒) → 循环
```

- **倒计时阶段**：玩家排队进入等待室
- **游戏阶段**：蛇移动、碰撞、得分
- **结束阶段**：显示胜利者，链上结算
- **胜利暂停**：最后幸存者出现后等待 24 tick（约 1.2 秒）才结束
- **预测场次**：系统会保留当前场、下一场和后续 future 槽位；已完成链上同步的场次会优先开放下注，未同步完成的场次保持锁定

---

## 四、蛇的机制

### 移动
- 四方向（上下左右），不能 180° 掉头
- 每 tick 移动 1 格

### 生命值 (HP)
- 初始 HP = 100
- 每 tick 消耗 1 HP（线上当前 50ms tick 下，不吃食物约 5 秒后饿死）
- 吃到食物：HP **恢复满** (100)

### 初始长度
- 出生时 3 节

### 成长
- 吃 1 个食物 → 长 1 节

---

## 五、碰撞与淘汰规则

| 场景 | 结果 |
|------|------|
| 撞墙 | 死亡 (`wall`) |
| 撞自己身体 | 死亡 (`self`) |
| 撞障碍物（竞技场） | 死亡 (`obstacle`) |
| 撞死蛇尸体 | 死亡 (`corpse`) |
| **头对头、同长度** | **双方都死** (`headon`) |
| **头对头、不同长度** | 短蛇死亡 (`eaten`)，长蛇存活（不吸收身体） |
| **头撞对方身体、自己更长** | 吞食：从碰撞点截断对方尾部，自己吸收被截断的节数，得分 +被吃节数 |
| **头撞对方身体、自己更短或等长** | 自己死亡 (`collision`) |

**死亡后**：
- 蛇尸体在场地上**持续闪烁直到比赛结束**（24 tick 为一个闪烁周期，循环反复），期间始终是**实体障碍**
- 被 "eaten" 的蛇尸体缩为 1 格（有效无残留）
- 竞技场中，死蛇尸体会**永久变成固体障碍物**（非 eaten 类型）

---

## 六、食物系统

### Performance 房
- 场上恒定 **5 个**食物

### Competitive 房（递减机制）
- 食物上限 = `ceil(剩余时间 / 30) - 1`

| 剩余时间 | 食物上限 |
|----------|---------|
| 180-151秒 | 5 |
| 150-121秒 | 4 |
| 120-91秒 | 3 |
| 90-61秒 | 2 |
| 60-31秒 | 1 |
| 30-0秒 | 0 |

- 食物不会出现在蛇身、已有食物、障碍物上
- 每个食物 = +1 分 + HP 恢复满

---

## 七、障碍物系统（仅竞技场）

- **生成频率**：每 80 tick（约 4 秒）生成一批
- **大小**：1-12 格（BFS 随机扩展出不规则形状）
- **生成位置**：避开边缘（2-27 范围）和已有障碍

**闪烁机制**：
1. 新生成 → **非实体**，闪烁 16 tick（约 0.8 秒）作为预警
2. 闪烁结束 → 变为**实体**，永久存在直到本局结束
3. 下一局开始时清空所有障碍物

---

## 八、胜负判定

| 场景 | 结果 |
|------|------|
| 多人局，只剩 1 人存活 | 该人获胜（等待 24 tick 约 1.2 秒确认） |
| 多人局，同一 tick 全部死亡 | 无胜者（平局） |
| 时间耗尽 | 按存活状态 + 死亡时间排名 |

**胜者判定**：时间耗尽时，存活蛇中**最长者**为胜者。

**排名逻辑**：
1. 存活蛇按：蛇长 ↓ → 分数 ↓ → HP ↓ → ID 字母序 ↑
2. 已死亡玩家按死亡时间倒序排列（越晚死排名越高）

## 九、Bot 类型与入场

### Normal Bot（普通 Bot）
- 系统自动创建，用于填充空位
- 使用内置 Flood-Fill AI 控制
- 无需付费，没有积分
- **可被 Agent/Hero Bot 踢出**

### Agent Bot（代理 Bot）
- 用户上传 JavaScript 代码，在 Worker 线程执行
- 创建时获得 **20 次免费试玩**
- 每场 Performance 比赛消耗 **1 credit**
- 链上注册（mint NFT）后 → **无限次**游玩

### Hero Bot（英雄 Bot）
- Agent Bot 的特殊类型（名字含 "HERO"）
- **最高优先级**：满员时可踢出 Normal Bot 入场
- 规则与 Agent Bot 相同

## 十、入场费与踢人规则

### Performance 房入场费（动态递增）
- 起始入场费：**0.01 ETH**
- 递增规则：当 60 个位置（6 间×10 人）全部被 ≥ 当前价格的 Agent 占满时，入场费 **+0.01 ETH**
- 费用持久化保存

### 踢人优先级
1. 先踢 **Normal Bot**（等待室中的优先）
2. 再踢 **入场费低于当前价格的 Agent/Hero**
3. 同价位不踢

### Credit 充值
- 0.01 ETH = 5 credits
- 充值公式：`packs = floor(amount / 0.01)`，每 pack 5 credits

## 十一、积分系统（Score）

### 获取方式

| 行为 | 积分 | 限制 |
|------|------|------|
| 每日签到 | 10 分 | 每天 1 次 |
| 连续签到 7 天 | 30 分（替代当天的 10 分） | 第 7 天触发，然后重置 |
| 参与比赛 | 5 分/场 | — |
| 比赛第 1 名 | 50 分 | - |
| 比赛第 2 名 | 30 分 | - |
| 比赛第 3 名 | 20 分 | - |
| 下注活跃 | 下注 1 USDC = 1 point（最低 1 分） | — |
| 注册 Bot | 200 分 | 一次性 |
| L1 推荐（直推） | 100 分 | 每推荐 1 人 |
| L2 推荐（间推） | 50 分 | 推荐人的推荐人 |

### 说明
- 当前排行榜按 **Epoch 积分** 计算；每个新 Epoch 会重新统计当期积分
- 用户的**累计总分**仍会保留，用于长期展示，不参与当前 Epoch 排名
- 签到等敏感操作需要钱包签名验证；下注积分通过链上 txHash 校验防刷
- 签名有效期 5 分钟
- 名次积分按真实名次逐个发放：前三里只要该名次对应注册 bot，就给该 bot owner 发分；不会因为另一个名次是 Normal Bot 而把整场前三积分全部取消

## 十二、Epoch/赛季系统

| 参数 | 测试网 | 主网 |
|------|--------|------|
| Epoch 时长 | 1 天 | **7 天** |
| 起始时间 | 2026-02-20 00:00 UTC | 同 |
| 计算公式 | `floor((now - origin) / (days × 86400000)) + 1` | 同 |

- 排行榜按 Epoch 分段统计
- 历史 Epoch 数据永久保留

## 十三、竞技场下注（Pari-Mutuel）

### 基本规则
- 使用 **USDC** 下注（链上智能合约）
- 下注窗口：当前以代码实现为准；只有已完成链上同步的场次开放下注，比赛一进入 `PLAYING` 就立即关闭下注
- 下注对象：选择某条蛇获胜
- 链上关闭和前端展示可能存在极短同步延迟，最终以链上状态为准

### 竞技场付费入场
- 竞技场付费入场费固定为 **0.001 ETH / 次**
- **任何钱包都可以为任何 Bot 购买竞技场席位**，不要求该钱包是 Bot owner
- 付费后，该 Bot 会被锁定进入指定的竞技场比赛场次

### 奖金分配
- 平台抽成：**5%**
- runner rewards：**5%**
- 剩余 **90%** 按名次分池后按下注比例分配给赢家下注者
  - 第 1 名奖池：90% 的 50%（总池 45%）
  - 第 2 名奖池：90% 的 30%（总池 27%）
  - 第 3 名奖池：90% 的 20%（总池 18%）
  - 不足 3 名赢家时按比例重分
- 无人下注赢家 → 该名次奖池按比例重分给有人下注的赢家名次
- 比赛取消 → 全额退款

### 结算
- 比赛结束后进入异步链上结算/取消队列；最终以链上 `settled` / `cancelled` 结果为准
- 调用 `settleMatch(matchId, winners[])` — 传入前 3 名 botId

### Runner Rewards（运行者奖励）
- 总池的 5% 按比赛名次分配给前 3 名 Bot 拥有者：第1名 3%，第2名 1.5%，第3名 0.5%
- 可通过 `claimRunnerRewards(botId)` 领取

## 十四、NFT 与 Bot 市场

### SnakeBot NFT（SNAKE）
- 每个链上注册的 Bot 铸造 1 个 NFT
- NFT = Bot 所有权证明

### 市场交易
- 挂单出售：Bot 托管到 BotMarketplace 合约
- 购买：ETH 支付
- 取消：随时下架
- 平台手续费：合约中设定

---

## 十五、推荐系统

- 最多 **2 层**深度（L1 直推 + L2 间推）
- 记录条件：被推荐人完成链上注册 Bot（需 txHash 证明）
- 每个用户只能有 1 个推荐人（不可更改）
- L1 推荐人得 100 积分，L2 推荐人得 50 积分

---

## 十六、Normal Bot AI 逻辑

内置 Flood-Fill 算法：
1. 评估 4 个方向的可达空间大小
2. 空间 < 自身长度 → 重罚（-10000 分）
3. 空间 < 自身长度 ×2 → 轻罚（-2000 分）
4. 靠近食物加分（短蛇权重更高）
5. 偏好中心位置
6. 惩罚靠墙（-30 分/边）
7. 躲避敌蛇头（-5000 分）

---

## 十七、反作弊与安全

### Bot 脚本沙箱
- 禁止：`require`, `import`, `eval`, `Function`, `process`, `global`, `__dirname`, `child_process`, `.constructor`, `.__proto__`
- AST 语法分析（acorn.js）

### 频率限制
| 类型 | 限制 |
|------|------|
| API 全局 | 120 次/分钟/IP |
| Bot 上传 | 10 次/分钟/IP |
| WebSocket 消息 | 20 条/秒/连接 |
| WebSocket 连接 | 10 个/IP |
| Admin 登录失败 | 指数退避（2s, 4s, 8s...最长 300s） |

### 交易防重放
- 入场 txHash 和推荐 txHash 去重
- 保留最近 5000 条记录

---

## 十八、比赛 ID 系统

- **内部 ID**：全局递增数字，用于链上结算
- **展示 ID**：字母+数字（如 A1, B3, P42）
  - A-F = Performance 房间
  - P = Competitive 竞技场
- 映射关系保留最近 500 条
- 支持通过展示 ID 查询/下注

---

## 十九、回放系统

- 录制时机：仅 PLAYING 状态，每 tick 录 1 帧（最多 2000 帧/场）
- 存储格式：JSON + gzip 压缩
- 保留时长：**7 天**，每 24 小时自动清理
- 内存索引：最新 200 场元数据
- 支持通过内部 ID 或展示 ID 查询
